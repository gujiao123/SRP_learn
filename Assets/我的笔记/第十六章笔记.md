# 📝 第16章 Render Scale 学习笔记

## 🎯 核心概念

**Render Scale = 让渲染分辨率和屏幕分辨率脱钩。**

- `renderScale < 1`：低分辨率渲染 → 放大到屏幕 → **省性能**（牺牲画质）
- `renderScale > 1`：高分辨率渲染 → 缩小到屏幕 → **超采样抗锯齿（SSAA）**（牺牲性能）
- `renderScale = 1`：正常渲染，不走缩放路径

> 类比：拍照时调低分辨率拍完再放大，银幕大小不变，但胶卷分辨率可以随时调。

## ⚠️ 关键问题与修复

### 问题1：`_ScreenParams` 和 buffer 尺寸不匹配

```
_ScreenParams = camera.pixelWidth（屏幕大小，不随 renderScale 变化）
实际 buffer = bufferSize（缩放后的大小）
→ 粒子扭曲特效的 screen UV 全部算错！
```

**修复**：引入 `_CameraBufferSize` 传给 GPU，Shader 里用 `_CameraBufferSize.xy`（即 1/w, 1/h）代替 `_ScreenParams`。

### 问题2：HDR 插值产生硬边

```
❌ 先插值再 ToneMap：
   像素A=0, 像素B=10 → 插值=5 → ToneMap → 0.99（硬白边）

✅ 先 ToneMap 再插值：
   像素A=0→0, 像素B=10→0.98 → 插值=0.49（正确的中间灰）
```

**修复**：在 bufferSize 分辨率下先做完 Color Grading + Tone Mapping（转 LDR），再用 FinalRescale Pass 插值缩放到屏幕。

### 问题3：Color Grading 色带

插值 HDR 颜色后经过 Color Grading 会产生虚假的中间色调（例如本不该出现的红色边缘）。

**修复**：同上——先 Color Grade 再缩放。

## 🔑 核心流程图

```
renderScale == 1 时：
  渲染 → PostFX(Bloom→ToneMap) → DrawFinal(Pass.Final) → 屏幕

renderScale != 1 时：
  渲染(bufferSize) → PostFX(Bloom→ToneMap)
       ↓
  Draw(Pass.Final) → 临时 LDR RT (bufferSize)    ← 先在小尺寸做完 ToneMap
       ↓
  DrawFinal(Pass.FinalRescale) → 屏幕            ← 再把 LDR 插值放大/缩小到屏幕
```

---

## 💡 注意事项

1. **renderScale 范围 0.1~2.0**：超过 2x 缩小采样时会跳过像素，质量反而变差
2. **±1% 容差**：`renderScale` 在 0.99~1.01 之间不触发缩放，避免浮点误差白白申请中间 buffer
3. **Scene 视图不缩放**：Scene 是编辑用的，缩放会导致选物体位置偏移
4. **Bloom ignoreRenderScale**：动态分辨率场景下建议开启，Bloom 不会跟着帧率变化闪烁
5. **Bicubic UpOnly 性价比最高**：放大时能改善质量，缩小时差距忽略不计
6. **无 PostFX 时没有 HDR 修复**：两步 Rescaling 只在 PostFX 激活时生效，无 PostFX 时假设没有 HDR
