# 📕 第四章：实时阴影 (Directional Shadows)

## 核心机制：Shadow Map

```
① 从灯光的视角渲染一次场景 → 只写深度（距灯光的距离）→ 存成 Shadow Map 贴图
② 正式渲染时，对于每个像素：
   把它的世界坐标转换到灯光空间 → 得到一个 UV 坐标 → 去 Shadow Map 取深度
   如果"当前深度 > 贴图里存的深度" → 说明前面有东西挡着 → 这个像素在阴影里
```

## 级联阴影 (Cascaded Shadow Maps, CSM)

**问题**：一张 Shadow Map 覆盖整个场景 → 近处精度极低，远处精度浪费。

**解决**：把摄像机视锥切成 N 块，近的块用小分辨率精细覆盖，远的块大范围低精度。

```
级联 0（最近，最小范围，最精细）
级联 1
级联 2
级联 3（最远，最大范围，最粗糙）

实现：把 4 个 Shadow Map 拼在一张大图集（Atlas）里的不同 Tile 上
```

## 关键数据结构

| 变量 | 含义 |
|---|---|
| `_DirectionalShadowMatrices` | 把世界坐标 → Tile 坐标的变换矩阵，每盏灯×每个级联 |
| `_CascadeCullingSpheres` | 每个级联的包围球，用于判断片元属于哪个级联 |
| `_DirectionalLightShadowData[i]` | `.x`=强度 `.y`=Tile起始 `.z`=法线偏移 `.w`=通道号（第六章加） |

## 主要问题与解法

| 问题 | 原因 | 解法 |
|---|---|---|
| Shadow Acne（自阴影锯齿）| 采样精度不足，物体给自己投影 | Normal Bias + Slope Scale Bias |
| 级联边界突变 | 不同级联精度巨大差异 | 级联混合（cascade blend）|
| 阴影在最远处消失 | 超出 Max Shadow Distance | 平滑淡出（FadedShadowStrength）|

## GetDirectionalShadowAttenuation 函数流程

```
进入函数
  ↓
材质没开 _RECEIVE_SHADOWS → return 1.0（不接受阴影）
  ↓
GetCascadedShadow()：
  找所属级联 → 计算 Normal Bias → 用矩阵变换到 Tile 坐标 → PCF 采样 → 如需混合 → lerp 下一级
  ↓
MixBakedAndRealtimeShadows()：
  根据 global/directional.strength 决定实时/烘焙的比例
  ↓
return shadow（0=全阴影，1=全亮）
```

# 📗 第五章：烘焙光照 (Baked Light)

## 核心思路

> "静态物体一辈子不动，提前把光照算好存成贴图，运行时直接读。"

**Lightmap 存的是：间接光（弹射光）的颜色**  
**不是阴影！** 阴影在第六章的 Shadow Mask 里。

## 三种灯光模式

| 模式 | 参与实时 | 参与烘焙 | 适合 |
|---|---|---|---|
| **Realtime** | ✅ | ❌ | 手电筒、点光源等动态灯 |
| **Baked** | ❌ | ✅ | 太阳、天光等永远不动的灯 |
| **Mixed** | ✅（直接光实时）| ✅（间接光烘焙）| 主光源（最常用）|

## 数据传输链路

```
Static 物体上面每个顶点都有两套 UV：
  第一套（TEXCOORD0）= 材质 UV（贴材质贴图用）
  第二套（TEXCOORD1）= Lightmap UV（指向图集里的位置）

顶点着色器里：
  lightMapUV = input.lightMapUV * unity_LightmapST.xy + unity_LightmapST.zw

片元着色器里（GI.hlsl）：
  gi.diffuse = SampleLightMap(lightMapUV)    ← 静态物体：读贴图
            + SampleLightProbe(surfaceWS);   ← 动态物体：读遮挡探针

Lighting.hlsl 里：
  float3 color = gi.diffuse * brdf.diffuse; ← 烘焙光 × 材质漫反射
```

## 动态物体的 GI：光照探针

```
探针 = 空间中一个点，烘焙时记录从各方向射来的光
存储格式 = 球谐函数（SH），27 个系数
插值方式 = 找到包围物体的四面体（由 4 个探针构成），用体积比（重心坐标）权重
```

## 两个重要大脑（互不通气！）

```
大脑 1：GPU 画师（我们的 Shader）
  → 认识 _BaseColor、_BaseMap
  → 把物体画到屏幕上

大脑 2：烘焙大师（Progressive Lightmapper）
  → 只认识 _Color、_MainTex、_Cutoff
  → 把光照计算结果存成贴图
```

**解决方案：Meta Pass**（`LightMode = "Meta"`）
- 两个大脑通信的翻译
- X 标志位 = 材质漫反射颜色（让色溢正确）
- Y 标志位 = 自发光颜色（让发光体照亮周围）

## MeshBall（GPU Instancing）的 GI 问题

```
DrawMeshInstanced → 没有 GameObject → Unity 自动探针系统绕过 → 全黑！

解决：
LightProbes.CalculateInterpolatedLightAndOcclusionProbes(positions, lightProbes, null);
block.CopySHCoefficientArraysFrom(lightProbes);
```

# 📘 第六章：阴影遮罩 (Shadow Masks)

## 核心问题

> "实时阴影超出 Max Shadow Distance 后消失，静态物体在远处没有阴影，很假！"

## Shadow Mask 贴图

- **RGBA 4 通道**，每个通道对应一盏 Mixed 灯
- 存的是：**静态物体之间的直接光遮挡关系**

```
贴图里的值含义：
  0.0（黑色）= 这个点被遮挡了（在阴影里）
  1.0（白色）= 这个点完全受光
  0.x（灰色）= 软阴影边缘（半影）
```

## 两种 Shadow Mask 模式

| 模式 | 关键词 | 静态物体阴影 | 动态物体阴影 | 混合方式 |
|---|---|---|---|---|
| **Distance Shadowmask** | `_SHADOW_MASK_DISTANCE` | 近处实时，远处切换烘焙 | 正常实时（不变） | `lerp(baked, realtime, global.strength)` |
| **Shadowmask（Always）**| `_SHADOW_MASK_ALWAYS` | 始终用烘焙 | 实时叠加 | `min(baked, realtime)` |

## 核心两个 strength

| | `global.strength` | `directional.strength` |
|---|---|---|
| 代表 | 像素到摄像机的距离衰减 | 灯光自身阴影强度 |
| 范围 | 近处=1.0，超出距离=0.0 | 面板 Shadow Strength 值 |
| 特殊值 | - | 负数 = 无实时投影体的信号 |

## 判断逻辑

```
if (directional.strength * global.strength <= 0.0) {
    // 两种情况满足：
    // 1. 摄像机太远，global.strength=0 → 乘积≤0 → 查烘焙
    // 2. 无实时投影体，directional.strength<0 → 乘积≤0 → 查烘焙
    shadow = GetBakedShadow(mask, channel, abs(strength));
} else {
    // 正常情况：实时采样 + 混合
    shadow = GetCascadedShadow(...);
    shadow = MixBakedAndRealtimeShadows(...);
}
```

## 多灯通道支持（Part C）

```
灯1 → occlusionMaskChannel=0 → shadows.r
灯2 → occlusionMaskChannel=1 → shadows.g
灯3 → occlusionMaskChannel=2 → shadows.b
灯4 → occlusionMaskChannel=3 → shadows.a

// 动态读取通道（不再硬编码 .r）：
shadow = mask.shadows[channel];
```

## 完整数据流

```
Unity 烘焙 → Shadow Mask 贴图 + occlusionMaskChannel 分配
    ↓
C# Shadows.cs：
  ReserveDirectionalShadows() 返回 Vector4(strength, tile, bias, maskChannel)
    ↓
Light.hlsl：
  _DirectionalLightShadowData[i].w → DirectionalShadowData.shadowMaskChannel
    ↓
GI.hlsl：
  SampleBakedShadows() → gi.shadowMask.shadows（全 4 通道）
    ↓
Lighting.hlsl：
  shadowData.shadowMask = gi.shadowMask（搬运数据）
    ↓
Shadows.hlsl：
  GetBakedShadow(mask, channel) → mask.shadows[channel]
  MixBakedAndRealtimeShadows(global, shadow, channel, strength) → 最终混合
```

# 🔗 三章的演进关系

```
第四章建立了：
  Shadow Map → 级联 → 采样函数 GetDirectionalShadowAttenuation

第五章建立了：
  GI struct（gi.diffuse + gi.shadowMask）
  烘焙管线 → Lightmap + 探针

第六章把两者缝合：
  gi.shadowMask → 传给 shadowData → 传给阴影计算
  远处：烘焙接替实时，阴影不再消失
  Always 模式：静态物体彻底解放实时阴影计算资源
```
